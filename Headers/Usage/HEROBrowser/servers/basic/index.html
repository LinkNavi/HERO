What is HERO?

HERO (High-Efficiency Reliable Object protocol) is a lightweight UDP-based protocol designed for fast, efficient communication. It combines the speed of UDP with reliability features like acknowledgments and retransmission.

Protocol Overview

HERO uses five main packet types for communication:

  • CONN (0) - Establish connection with server
  • GIVE (1) - Send data payload to recipient
  • TAKE (2) - Request resource from server
  • SEEN (3) - Acknowledge packet receipt
  • STOP (4) - Gracefully close connection

Getting Started

Creating a HERO server is simple:

#include "HERO.h"

int main() {
    HERO::HeroWebServer server(8080);
    
    while (server.isRunning()) {
        server.serve();
    }
    
    return 0;
}

The HERO protocol and browser are written in modern C++17 with minimal dependencies. The browser uses SDL2 for rendering and supports cross-platform development.

Performance

HERO's UDP foundation provides:

  • Latency under 5ms on local networks
  • Throughput up to 1 Gbps on modern hardware
  • Low CPU overhead compared to TCP
  • Minimal memory footprint
